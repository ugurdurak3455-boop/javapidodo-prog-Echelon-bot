# Echelon UI/UX Redesign Plan

## 1. Understanding Summary
* **Что мы строим:** Комплексное улучшение UI/UX для трёх компонентов Echelon: Telegram-бота, веб-дашборда администратора и десктопного лаунчера.
* **Зачем это нужно:** Сделать взаимодействие с системой более приятным, логичным и современным (улучшить оформление, кнопки, фильтры, убрать лишнее).
* **Для кого:** Закрытое сообщество (до 500 человек).
* **Ключевые ограничения:** База данных SQLite, дашборд работает через `http.server` (без тяжелых JS/CSS фреймворков-сборщиков, только CDN), лаунчер на `customtkinter`.
* **Явные "не-цели":** Мы НЕ делаем сложную систему прав, НЕ пишем отдельный бекенд для API, НЕ используем React/Vue с пайплайнами сборки.

## 2. Assumptions (Допущения)
1. **Производительность:** `http.server` с SQLite справится с нагрузкой, так как дашборд используется редко и в основном только администратором.
2. **Безопасность:** Защита дашборда на уровне Basic Auth достаточна. Бот открыт, но не афишируется (доступ "только для своих").
3. **Окружение:** Лаунчер запускается локально на Windows, который стабильно поддерживает `customtkinter`.

## 3. Decision Log
1. **Подход к архитектуре дизайна:** 
   * *Решение:* "Полировка ядра" (Polished Core) - улучшение текущих компонентов без смены архитектуры.
   * *Альтернативы:* Создание SPA дашборда (слишком сложно, оверхед) или перенос всего в Telegram-бота (отказ от дашборда, плохо для визуализации).
   * *Почему выбрано:* Минимальные риски поломки текущего стабильного кода, быстрая реализация, соблюдение принципа YAGNI.
2. **Навигация в боте:**
   * *Решение:* Переход от ручного ввода команд к постоянному Inline/Reply меню с кнопками "Назад".
   * *Альтернативы:* Оставить как есть (только `/команды`).
   * *Почему выбрано:* Кардинально повышает удобство использования с телефона.
3. **Интерактивность дашборда:**
   * *Решение:* Оставить Vanilla JS для поиска/фильтрации, но добавить кнопку "Сбросить фильтры", sticky headers и CSS progress bars.
   * *Альтернативы:* Внедрить Alpine.js или React.
   * *Почему выбрано:* Достаточно для текущих нужд без усложнения кодовой базы.

## 4. Final Design Specification

### Часть 1: Дизайн Telegram-бота 🤖
* **Главное меню:** Вызывается по `/start`. Содержит кнопки: `[📋 Мои фильтры]`, `[🏢 Города]`, `[📱 Модели]`, `[⭐ Избранное]`.
* **Навигация:** Вложенные меню имеют кнопку `[🔙 Назад]`.
* **Выбор:** Inline-клавиатуры для моделей/городов используют галочки: `[✅ Москва]`, `[❌ Казань]`.
* **Уведомления:** Цена (жирным), медиана (курсивом), выгода (цветной эмодзи/индикатор). Кнопки `[🔗 Открыть на Avito]` и `[⚠️ Пожаловаться]`.

### Часть 2: Дизайн Веб-дашборда 🌐
* **Визуал:** Палитра в стиле Tailwind Slate/Zinc. Плавные hover-эффекты на карточках.
* **KPI "Точность":** Добавлен CSS-прогресс-бар для визуализации процента точности.
* **Таблицы:** Сделаны Sticky Headers.
* **Поиск:** Добавлена кнопка очистки фильтров. При отсутствии результатов показывается сообщение "Empty state".

### Часть 3: Десктопный Лаунчер 🖥️
* **Цвета кнопок:** Зеленый для "Запустить", приглушенный красный для "Остановить".
* **Консоль логов:** Добавлен чекбокс/кнопка "Автопрокрутка (Вкл/Выкл)". Выровнены префиксы строк логов (`[Бот]`, `[Веб]`) для удобного чтения колонкой.
* **Сайдбар:** Настройки (Демо/Авито, Браузер) сгруппированы в блок "Настройки парсера".
